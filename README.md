# 🚀 Concorrência em Bancos de Dados  
### O que todo desenvolvedor precisa dominar

> 🎤 Palestra apresentada no evento Coda.CE

Material oficial da palestra abordando conceitos fundamentais de concorrência em bancos de dados com foco em **SQL Server**.

---

## 📚 Conteúdo

Este repositório contém:

- 📄 Slides da apresentação (PDF)
- 💻 Scripts utilizados nos hands-on
- 🧪 Exemplos práticos de:
  - Blocking
  - Deadlock
  - Monitoramento de concorrência

---

## 🧠 O que você vai aprender

- Importância da concorrência em sistemas
- O que são locks e como funcionam
- Tipos de locks (S, X, U, Intent, etc.)
- Recursos que podem ser bloqueados
- Matriz de compatibilidade de locks
- O que é blocking e como identificar
- Níveis de isolamento de transações
- Deadlocks: como acontecem e como evitar

---

## 🔬 Hands-on (parte prática)

Você encontrará scripts para simular e entender na prática:

- ✔️ Cenários de blocking
- ✔️ Monitoramento de sessões e locks
- ✔️ Simulação de deadlock
- ✔️ Estratégias para análise e resolução

---

## ⚠️ Por que isso é importante?

Problemas de concorrência são responsáveis por:

- ❌ Lentidão intermitente
- ❌ Timeout em aplicações
- ❌ Deadlocks inesperados

Dominar esses conceitos é essencial para qualquer desenvolvedor ou profissional de dados que trabalha com sistemas críticos.

---

## 🛠️ Como utilizar este material

1. Baixe o PDF da apresentação
2. Execute os scripts em um ambiente SQL Server
3. Reproduza os cenários demonstrados
4. Analise o comportamento das transações

💡 Dica: utilize múltiplas sessões no SSMS para simular concorrência
